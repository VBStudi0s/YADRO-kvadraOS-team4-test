# Media files searcher

## Формулировка задания

Основное задание:

Написать приложение на языке С++ под Linux, которое через равные интервалы времени будет обходить домашний каталог и находить в нем мультимедийные файлы (изображения, аудио, видео) и формировать файл в домашнем каталоге .media_files в формате json следующего вида:
{ "audio": [ "111.mp3", "222.wav" ], "video": [ "333.mpg" ] "images": [ "444.jpeg", "555.png" ] }

К коду должен прилагаться скрипт любой системы сборки (make, cmake, etc., но не shell-скрипт).

Код должен быть выложен на git-репозиторий, содержать build-скрипт (make, cmake, etc) и readme-файл с инструкцией по построению и запуску.

Решение присылать в виде ссылки на репозиторий и sha-commit'а/tag'а.

Дополнительные задания (опционально):

• Периодичность с которой отслеживается каталог и путь до каталога настраиваются при запуске

• Результирующий json-файл не создается на файловой системе, вместо этого он доступен через HTTP по адресу http://localhost:1234/media_files через GET-запрос

## Описание решения

Многопоточное приложение для рекурсивного анализа каталогов, поиска мультимедиа файлов и выдачи их списка по HTTP-запросу.
Программа принимает корневой каталог для анализа и период обхода. Раз в указанный период в отдельном потоке стартует процедура обхода.
также поднят сервер для прослушивания порта 1234 и выдачи по запросу актуального списка файлов.

## Используемые библиотеки

В проекте используются следующие (интегрированы в проект) header-only библиотеки:
- nlohmann-json
- httplib

## Сборка и запуск

Для сборки пропишите команду
```
make build
```

Для запуска приложения используйте:
```
./build/media_list --period <Период обхода> --path <Корневая директория для обхода>
```

Или настройте параметры в `Makefile` и используйте

```
make run
```

### Пример работы

```
~: ./build/media_list --period 1000 --path ./test
~: curl -X GET localhost:1234/media_files
{"audio":["hello.wav","a.mp3"],"images":["ping.png"],"video":[]}
```
